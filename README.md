# Video Game Guides

A collection of interactive, single-file HTML guides for video games — and the
custom Claude Code skill that builds them.

## What this project is for

The goal of this repo is to produce **trackable achievement / completion
guides** that fuse two things into one page:

- a **chronological, turn-by-turn walkthrough** (the route through the game), and
- an **achievement / trophy / completion checklist**, where each achievement is
  flagged inline at the exact step where you can earn it.

Each finished guide is a self-contained HTML file with a sticky progress
tracker, a table of contents, missable warnings, embedded maps and screenshots,
and a checklist that saves your progress in the browser via `localStorage`.
Open it, follow the route top to bottom, and tick things off as you go.

These are aimed especially at **RetroAchievements**-style 100% runs, but the
skill works for any game and any walkthrough source.

## How the guides get made

Guides are generated by the **`achievement-guide-builder`** skill, which lives in
`.claude/skills/achievement-guide-builder/`. It's invoked through Claude Code:
ask for "a guide / walkthrough / achievement guide / 100% guide" for a game and
the skill drives the process.

The skill is deliberately strict so the output stays trustworthy:

1. **Your sources are the truth.** It builds only from sources you provide
   (links or uploaded files). It may *propose* extra sources, but only uses them
   once you approve.
2. **It invents nothing.** Every place, NPC, item name, count, input, and
   threshold comes from a source and is reproduced exactly; only connecting prose
   is lightly reworded. Gaps are marked and asked about, never guessed.
3. **No step is too small.** The route aims to be exhaustive — which door, which
   direction, who to talk to — capped only by what the source actually says.
4. **Maps and screenshots are included** at the step they illustrate.
5. **It asks instead of deciding.** Source choice, ordering, missable
   classification, theming, and gap resolution are all surfaced to you.

Generation runs through four gates, stopping for your sign-off at each: **(1)
Sources → (2) Outline → (3) Sample chapter → (4) Full build.**

## Repository layout

```
.
├── README.md
├── ocarina-of-time/                      # finished guides for a game
│   ├── ocarina-of-time-ra-guide-v01.html
│   └── ocarina-of-time-ra-guide-v02.html
└── .claude/skills/achievement-guide-builder/
    ├── SKILL.md                          # full skill instructions & workflow
    ├── build.py                          # guide.json → one self-contained HTML
    ├── fetch_images.py                   # localizes remote images
    ├── assets/                           # the canonical CSS/JS kit
    └── references/schema.md              # the guide.json data contract
```

Each game gets its own folder of HTML guides. Versions (`v01`, `v02`, …) let a
guide be refined over time while keeping older builds around.

## Building a guide manually

The skill normally handles this, but the underlying pipeline is just two
scripts driven by a `guide.json` authored against
`.claude/skills/achievement-guide-builder/references/schema.md`:

```bash
cd .claude/skills/achievement-guide-builder

# 1) Localize images into the output's asset folder (best-effort).
python fetch_images.py guide.json --img-dir output/assets/img

# 2) Render the guide into one self-contained HTML file.
python build.py guide.json -o output/<slug>.html
```

`build.py` embeds the CSS/JS kit and derives the tracker, TOC, and all counts —
it never edits wording. Keep a guide's `slug` stable across rebuilds so players
don't lose saved progress.
